Overview

Multi-step registration form for apartment complex managers to onboard their properties onto the platform. Provisions user accounts via AWS Cognito and stores property data in DynamoDB through AppSync.

Registration Flow

StepScreenDetails
1Personal InfoName, email (duplicate check), phone, password
2Complex InfoComplex name, address, unit count, contact info, image upload
3Parking ConfigTotal spots, tenant/guest/admin/handicap allocation with total validation
4ReviewConfirmation of all data. On submit: creates User, signs up in Cognito, creates Tenant
5Verify EmailEnter Cognito confirmation code
6CompleteSuccess message with Stripe payment redirect

Orchestration with Rollback

The submit handler on Step 4 performs a multi-step orchestration:

  • Create User record in AppSync (DynamoDB)
  • Sign up user in AWS Cognito
  • Update User with Cognito ID
  • Create Tenant record
  • Post to Google Sheets (optional CRM logging)
  • If any step fails, all previous operations are rolled back

Validation

  • Name: Letters only, sanitized on input
  • Email: Regex validated, checked for existing accounts via GraphQL query
  • Phone: 10-15 digits, optional + prefix
  • Password: Min 8 characters, must match confirmation
  • Parking: Total allocated cannot exceed total spots
